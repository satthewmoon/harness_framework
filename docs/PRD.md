## 7. 에러 케이스 및 엣지 케이스

> 이 섹션은 필수다. AI가 에러 처리 없이 Happy Path만 구현하는 것을 방지한다.

### 7-1. 입력 에러
| 케이스 | 예상 동작 |
|--------|-----------|
| 필수 환경변수 누락 | 시작 시 즉시 에러 메시지 출력 + `sys.exit(1)` |
| `.env` 파일 없음 | 경고 후 OS 환경변수로 fallback. 없으면 에러. |
| 잘못된 형식의 입력값 | 명확한 에러 메시지 후 종료 (스택트레이스 숨김) |
| 입력 값 범위 초과 | {프로젝트별 정의 — 예: URL이 아닌 문자열 입력 시} |

### 7-2. 네트워크·외부 API 에러
| 케이스 | 예상 동작 |
|--------|-----------|
| HTTP 타임아웃 | 최대 N회 재시도 (지수 백오프). 모두 실패 시 에러 로그 |
| HTTP 4xx (클라이언트 에러) | 재시도 없이 즉시 에러 기록. 원인 로그 포함. |
| HTTP 5xx (서버 에러) | N회 재시도. 모두 실패 시 에러 로그 |
| 네트워크 단절 | 재시도. 복구 불가 시 에러 로그 + 알림 (설정 시) |
| API 응답 형식 불일치 | 파싱 에러 기록. 해당 항목 스킵 또는 전체 중단. |
| API 인증 만료 (401) | 즉시 중단. 자격증명 재설정 안내 메시지 출력. |
| Rate limit (429) | `Retry-After` 헤더 값만큼 대기 후 재시도. |

### 7-3. 데이터·파일 에러
| 케이스 | 예상 동작 |
|--------|-----------|
| DB 연결 실패 | 재시도 N회. 실패 시 에러 로그 + 종료. |
| DB 쓰기 충돌 (unique 위반 등) | 해당 항목 스킵 (upsert 가능 시 upsert) |
| 파일 읽기 권한 없음 | 에러 메시지 출력 + 종료. 경로 안내 포함. |
| 디스크 공간 부족 | 에러 메시지 출력 + 종료. |
| 빈 데이터 / 결과 없음 | 에러가 아님. INFO 로그 출력 후 정상 종료. |
| 예상보다 많은 데이터 | {예: 최대 N개로 제한. 초과분은 다음 실행으로 이월.} |
